# JPostman Kubernetes Ansible Dev Setup

This setup uses two playbooks:

```text
src/test/resources/
├── install-kubernetes-tools.yml   # optional first-time sudo setup
├── setup-kubernetes-dev.yml       # no sudo; creates kind cluster and test resources
└── cleanup-kubernetes-dev.yml     # no sudo; deletes kind cluster
```

## 1. Install Ansible Kubernetes collection

Run once:

```bash
ansible-galaxy collection install kubernetes.core
```

## 2. Optional first-time tools install

Run this only if Docker, kubectl, kind, or Python Kubernetes libraries are missing:

```bash
cd src/test/resources/
ansible-playbook -i localhost, -c local install-kubernetes-tools.yml --ask-become-pass
```

This playbook uses:

```yaml
become: true
```

because installing packages and writing to `/usr/local/bin` require sudo.

After it adds your user to the `docker` group, restart your shell or WSL session.

## 3. Run the no-sudo Kubernetes setup

After tools are installed, use this command for normal setup:

```bash
ansible-playbook -i localhost, -c local setup-kubernetes-dev.yml
```

It assumes these commands already work without sudo:

```bash
docker version
docker ps
kubectl version --client=true

kind get clusters
kubectl config get-contexts
kubectl get namespaces
kubectl --context kind-jpostman-dev get serviceaccount -n jpostman-dev
kubectl --context kind-jpostman-dev get role -n jpostman-dev
kubectl --context kind-jpostman-dev describe role jpostman-kubernetes-test -n jpostman-dev
kubectl --context kind-jpostman-dev get rolebinding -n jpostman-dev
kubectl --context kind-jpostman-dev get configmap -n jpostman-dev
kubectl --context kind-jpostman-dev get secret -n jpostman-dev
kubectl --context kind-jpostman-dev get pods -n jpostman-dev
kubectl get pods -A
```

If `docker ps` gives permission denied, restart your shell/WSL after running the install playbook.

## What setup creates

```text
kind cluster:      jpostman-dev
context:           kind-jpostman-dev
namespace:         jpostman-dev
service account:   jpostman-test-runner
role:              jpostman-kubernetes-test
role binding:      jpostman-kubernetes-test-binding
config map:        jpostman-test-config
secret:            jpostman-test-secret
pod:               jpostman-test-pod
```

## Generated test files

The setup writes:

```text
src/test/resources/jpostman-kind-kubeconfig.yaml
src/test/resources/kubernetes-local.properties
```

Example properties:

```properties
# Local Kubernetes integration test settings generated by Ansible.

KUBECONFIG=jpostman-kind-kubeconfig.yaml
KUBE_CONFIG_MAP_NAME=jpostman-test-config
KUBE_SECRET_NAME=jpostman-test-secret
KUBE_CONTEXT=kind-jpostman-dev
KUBE_NAMESPACE=jpostman-dev
KUBE_POD_NAME=jpostman-test-pod
KUBE_SERVICE_ACCOUNT_NAME=jpostman-test-runner
```

## Cleanup

```bash
ansible-playbook -i localhost, -c local cleanup-kubernetes-dev.yml
```
